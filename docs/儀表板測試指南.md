## 🐛 疑難排解

### 問題 1: 一直顯示「載入中...」

**可能原因**：
1. Google Sheet 未設為公開
2. CSV 連結無法存取
3. 網路問題

**解決方法**：
1. 按 **F12** 開啟瀏覽器開發者工具
2. 切換到 **Console** 分頁
3. 查看錯誤訊息
4. 常見錯誤：
   - `Failed to fetch` → Google Sheet 未公開
   - `CORS error` → Google Sheet 未公開
   - 其他網路錯誤

**快速測試**：
- 在瀏覽器新分頁開啟步驟 2 的 CSV 連結
- 如果無法開啟 → Google Sheet 權限問題

### 問題 2: 某些區塊沒有資料

**可能原因**：
- 對應的工作表沒有資料
- 公式沒有正確執行

**解決方法**：
1. 回到 Google Sheet
2. 檢查「打卡統計」工作表是否有資料
3. 檢查「每日亮點牆」工作表是否有資料
4. 如果沒有資料，執行 Apps Script 的 `generateTestData` 產生測試資料

### 問題 3: 連續打卡天數都是 0

**可能原因**：
- Apps Script 的 `calculateConsecutiveDays` 函數未執行
- 或使用優化版，需要手動執行 `updateAllConsecutiveDays`

**解決方法**：
1. 開啟 Google Sheet
2. 點擊「擴充功能」→「Apps Script」
3. 函數下拉選單選擇 `updateAllConsecutiveDays`（優化版）
4. 或 `generateTestData` 重新產生資料
5. 點擊「執行」
6. 回到 Sheet 檢查 C 欄是否有數字
7. 重新整理儀表板

### 問題 4: 今日打卡人數/打卡率顯示不準

**說明**：
- 目前這兩個數字是**模擬數據**（假設 80% 打卡率）
- 要顯示真實數據需要從「表單回應」計算今日打卡人數
- 可以在未來版本中加入此功能

### 問題 5: 個人查詢的 7 天記錄不準確

**說明**：
- 目前 7 天記錄是**模擬數據**（隨機產生）
- 要顯示真實數據需要從「表單回應」查詢該學員的記錄
- 可以在未來版本中加入此功能

### 修改課程開始日期

如果你的課程不是 2026-01-03 開始，請修改：

1. 開啟 `dashboard.html`
2. 找到第 507 行：
   ```javascript
   const COURSE_START_DATE = new Date('2026-01-03');
   ```
3. 改為你的實際開課日期：
   ```javascript
   const COURSE_START_DATE = new Date('2025-01-13');
   ```
4. 儲存檔案
