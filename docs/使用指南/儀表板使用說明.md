## 📋 快速啟用步驟

### 步驟 1: 取得工作表 gid

1. **開啟你的 Google Sheet**
   https://docs.google.com/spreadsheets/d/1C1t_hUTFDaQlxS4np8NiieyOiwPdG0ZGK_zn2wlHDSA/edit

2. **點擊「打卡統計」工作表**（底部標籤）

3. **看網址最後的數字**
   例如：`#gid=123456789`
   → 複製 `123456789` 這個數字

4. **點擊「每日亮點牆」工作表**

5. **再次看網址最後的數字**
   → 複製這個數字

---

### 步驟 2: 確認 Google Sheet 共用設定

這一步非常重要，否則儀表板無法讀取資料！

1. 在 Google Sheet 點擊右上角「**共用**」按鈕
2. 在「一般存取權」區域，點擊「限制」
3. 改為「**知道連結的任何人**」
4. 權限設為「**檢視者**」
5. 點擊「完成」

---

### 步驟 3: 修改 dashboard.html

1. **用文字編輯器（VS Code、記事本等）開啟 `dashboard.html`**

2. **找到第 391-392 行**（在 `<script>` 標籤中）：

```javascript
// TODO: 請替換為實際的 gid
const STATS_GID = 'YOUR_STATS_GID';      // 打卡統計工作表的 gid
const HIGHLIGHTS_GID = 'YOUR_HIGHLIGHTS_GID';  // 每日亮點牆工作表的 gid
```

3. **替換為你的實際 gid**：

```javascript
// 範例：假設你的 gid 分別是 123456 和 789012
const STATS_GID = '123456';      // 打卡統計工作表的 gid
const HIGHLIGHTS_GID = '789012';  // 每日亮點牆工作表的 gid
```

4. **（可選）修改課程開始日期**（第 397 行）：

```javascript
const COURSE_START_DATE = new Date('2026-01-03');  // 改為你的實際開課日期
```

5. **儲存檔案**

---

### 步驟 4: 測試儀表板

1. **用瀏覽器開啟 `dashboard.html`**
   - 雙擊檔案，或
   - 拖曳到瀏覽器視窗

2. **檢查是否正常顯示**：
   - ✅ 整體進度看板顯示數字
   - ✅ 連續打卡王排行榜顯示學員
   - ✅ 每日亮點牆顯示亮點
   - ✅ 個人查詢可以選擇學員

3. **如果顯示「載入中...」一直不消失**：
   - 按 F12 開啟瀏覽器開發者工具
   - 切換到「Console」分頁
   - 查看錯誤訊息
   - 常見問題：
     - gid 不正確
     - Google Sheet 未設定為公開
     - 網路問題

## 🚀 部署方式

### 方式 1: 本地使用（最簡單）
- 直接用瀏覽器開啟 `dashboard.html`
- 適合個人或小組使用

### 方式 2: 雲端託管（推薦給學員）

#### 使用 GitHub Pages（免費）：

1. **建立 GitHub 帳號**（如果沒有）

2. **建立新的 Repository**
   - 名稱：`5week-reflection-dashboard`
   - 設定為 Public

3. **上傳 `dashboard.html`**
   - 可直接拖曳上傳

4. **啟用 GitHub Pages**
   - Settings → Pages
   - Source: Deploy from a branch
   - Branch: main → / (root)
   - 點擊 Save

5. **取得公開連結**
   - 例如：`https://your-username.github.io/5week-reflection-dashboard/dashboard.html`
   - 分享給學員即可

#### 使用 Netlify Drop（最簡單，免費）：

1. 前往 https://app.netlify.com/drop
2. 直接拖曳 `dashboard.html` 到網頁
3. 立即獲得公開連結
4. 分享給學員

## 🐛 常見問題

### Q1: 儀表板一直顯示「載入中...」？

**檢查項目**：
1. ✅ Google Sheet 是否設定為「知道連結的任何人都能檢視」
2. ✅ `STATS_GID` 和 `HIGHLIGHTS_GID` 是否正確
3. ✅ 網路連線是否正常

**測試方法**：
- 直接在瀏覽器開啟 CSV 連結，看是否能下載
- 例如：`https://docs.google.com/spreadsheets/d/1C1t_hUTFDaQlxS4np8NiieyOiwPdG0ZGK_zn2wlHDSA/export?format=csv&gid=YOUR_GID`

### Q2: 顯示的資料不正確？

**可能原因**：
- Google Sheet 的欄位順序不正確
- 公式沒有正確執行

**檢查**：
- 參考 `URGENT-FIX.md` 確認公式正確
- 確認「打卡統計」和「每日亮點牆」的資料格式正確

### Q3: 今日打卡人數不準確？

**說明**：
- 目前「今日打卡人數」是模擬數據（80% 打卡率）
- 要顯示真實數據，需要從「表單回應」工作表計算
- 可以在未來版本中加入此功能

### Q4: 個人查詢的 7 天記錄不準確？

**說明**：
- 目前 7 天記錄是模擬數據
- 要顯示真實數據，需要從「表單回應」查詢該學員的記錄
- 可以在未來版本中加入此功能
